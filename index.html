Selected case studies

How the production results were achieved

Measured on production builds with Unity Profiler, Memory Profiler snapshots and Addressables Build Report.

01

Loading and Addressables

Reduced scene dependencies and up-front world creation

Problem
Hub and mission loads pulled oversized dependency sets while enemies, loot and interactable objects were all created before the player needed them.
Solution
Separated scene-owned content from shared assets, audited bundle dependencies and deferred world-object creation according to distance from the player.
Result
Hub loading dropped from 30s to 18s; mission loading from 47s to 23s.
02

Memory lifecycle

Eliminated unintended mission retention and duplicated content

Problem
Legacy mission scripts remained reachable through static events, closures and singleton services; the undisposed ECS context retained the full mission graph. Bundles also duplicated shaders and assets.
Solution
Used Memory Profiler snapshots to trace retention paths, refactored lifecycle cleanup and subscriptions, disposed the mission context and removed duplicated bundle content.
Result
Menu memory fell from 3-4 GB to about 1.8 GB, hub from 7-8 GB to 4-4.5 GB, missions from 10-12 GB to 6-8 GB.
03

Combat performance

Removed duplicate combat evaluation and pathfinding work

Problem
Ability simulation repeated the same evaluation and allocations; enemy turns repeatedly recalculated abilities and pathfinding, producing severe frame stalls.
Solution
Removed duplicate evaluation passes and redundant path searches, then replaced allocation-heavy temporary collections with custom thread-safe pools.
Result
Ability simulation no longer collapsed from about 110 to 15 FPS; enemy turns no longer fell from 80-100 to 1-5 FPS.
